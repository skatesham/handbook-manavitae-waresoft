    🎯 Diferenças Principais

    v1.0: Guia Simplificado

    • ✅ Foco em Angular 20 + PrimeNG + Tailwind
    • ✅ Regras claras e diretas
    • ✅ Checklist de PR
    • ❌ Abordagem única (não escalável)
    • ❌ Poucos exemplos práticos
    • ❌ Sem guias de decisão

    v2.0: Guia Polimórfico

    • ✅ Tudo da v1.0 +
    • ✅ 3 níveis de escala (Pequeno/Médio/Grande)
    • ✅ 4 exemplos completos de código real
    • ✅ 5 tabelas de decisão rápida
    • ✅ Guia de decisão rápida para dúvidas comuns
    • ✅ Métricas objetivas de qualidade
    • ✅ Roadmap de evolução em 3 fases
    • ✅ Índice navegável com links

    📋 Conteúdo Novo na v2.0

    1. Índice Rápido (Novo)

    Navegação rápida com links internos para todas as seções.

    2. Padrões de Escala (Expandido)

    v1.0: Regras únicas para todos
v2.0: Critérios específicos por tamanho de projeto

    Exemplo: - Pequeno: Services apenas se lógica > 30 linhas - Médio: Um serviço por domínio - Grande: Separar Application/Domain/Infrastructure

    3. Guia de Decisão Rápida (Novo)

    Respostas imediatas para: - Onde colocar meu código? - Quando extrair? - Signal, Getter ou Computed?

    4. Exemplos Completos (Expandido)

    v1.0: 2 exemplos básicos
v2.0: 4 exemplos completos
  - Componente Dumb (Shared UI)
  - Componente Smart (Feature Page)
  - Serviço de Domínio
  - Utilitário Puro

    5. Métricas de Qualidade (Novo)

    Limites objetivos por tamanho de projeto: - Linhas/Componente - Linhas/Serviço - Linhas/Template - Níveis/Template - Métodos/Classe - Parâmetros/Método

    6. Sinais de Alerta (Novo)

    Indicadores de quando refatorar: - Arquivo > 500 linhas - Método > 50 linhas - Complexidade ciclomática > 10 - Imports > 20 - Duplicação > 3 vezes

    7. Evolução da Arquitetura (Novo)

    Roadmap em 3 fases: - Fase 1: MVP (1-3 meses) - Fase 2: Produto (3-12 meses) - Fase 3: Escala (12+ meses)

    🎯 Casos de Uso

    Quando usar v1.0?

    • ❌ Não usar mais
    • ✅ Referência histórica apenas

    Quando usar v2.0?

    • ✅ Sempre - substitui completamente v1.0
    • ✅ Novos desenvolvedores
    • ✅ Code review
    • ✅ Decisões arquiteturais
    • ✅ Planejamento de features
    • ✅ Refatorações

    📊 Impacto Esperado

    Antes (v1.0)

    • ⚠️ Dúvidas sobre quando criar arquivos
    • ⚠️ Inconsistência entre projetos pequenos/grandes
    • ⚠️ Poucos exemplos práticos
    • ⚠️ Sem métricas objetivas

    Depois (v2.0)

    • ✅ Decisões rápidas com guias específicos
    • ✅ Arquitetura adapta-se ao tamanho do projeto
    • ✅ 4 exemplos completos de referência
    • ✅ Métricas objetivas para code review

    🚀 Migração

    Para Projetos Existentes

    1. Não precisa refatorar - v2.0 é compatível com v1.0
    2. Use como referência para novas features
    3. Aplique gradualmente os novos padrões
    4. Consulte "Padrões de Escala" para seu tamanho de projeto

    Para Novos Projetos

    1. Comece com "Pequeno" (Fase 1: MVP)
    2. Evolua para "Médio" quando necessário (Fase 2: Produto)
    3. Escale para "Grande" apenas se justificado (Fase 3: Escala)

    ✅ Conclusão

    v1.0: Guia Simplificado

    • ✅ Bom para começar
    • ❌ Limitado para escalar

    v2.0: Guia Polimórfico

    • ✅ Completo: Cobre todos os cenários
    • ✅ Escalável: Do MVP ao Enterprise
    • ✅ Prático: Exemplos reais
    • ✅ Navegável: Índice e estrutura
    • ✅ Evolutivo: Cresce com o projeto

    Recomendação: Use v2.0 como referência única. Mantenha v1.0 apenas para histórico.
